# 폰덱스(Phondex) 브랜드 마스터 플랜 — "휴대폰의 모든 결정"

> 작성일 2026-07-03. 목표: 기종별 결정 사전(현재)을 넘어, **휴대폰의 생애주기 전체(사기 전 → 쓰는 중 → 팔 때)를 커버하는 국내 1위 휴대폰 결정 플랫폼**으로 확장.
> 원칙: 스펙 나열·리뷰 감상평은 다루지 않는다. **숫자·날짜·출처가 있는 "결정 데이터"만** 다룬다. 이것이 브랜드의 해자.

---

## 0. 브랜드 정의

### 포지셔닝 한 줄
> **"폰에 대한 모든 결정, 폰덱스에서 30초."**

| 축 | 경쟁자 | 폰덱스의 차별점 |
|---|---|---|
| 스펙 비교 | GSMArena, 다나와 | 스펙이 아니라 **판정**(사도 될까/팔아야 할까)을 준다 |
| 리뷰 | 유튜버, 블로그 | 주관 배제. 출처·기준일 명시된 데이터만 |
| 시세 | 번개장터/당근 검색 | 검색이 아니라 **시계열 + 적정가 판정 + 타이밍** |
| 커뮤니티 | 뽐뿌, 클리앙 | 소음 제거. 검증된 이슈만 편집 게재 |

### 브랜드 지켜야 할 3원칙 (모든 신규 기능의 리트머스)
1. **결정에 기여하는가?** — 안 하면 안 만든다 (예: 벤치마크 점수 나열 ✗)
2. **출처·기준일·신뢰등급을 달 수 있는가?** — 못 달면 게재 안 함
3. **자동 갱신 또는 저비용 갱신이 가능한가?** — 썩는 콘텐츠는 부채

### 타깃 페르소나 (4명)
- **P1 구매 고민러**: "S24 지금 사도 돼? 중고가 나아?" → 현재 사이트의 코어
- **P2 보유자**: "내 폰 언제까지 쓸 수 있지? 배터리 갈까 바꿀까?" → **미개척, 최대 확장 기회**
- **P3 판매 예정자**: "언제 팔아야 손해 안 보지?" → 시세 데이터 재활용으로 저비용 진입
- **P4 부모/선물러**: "애한테/부모님께 뭘 사주지?" → 추천 마법사로 커버

## 2. 제품 확장 — 생애주기 4단계 × 기능 매트릭스

### 🅐 사기 전 (Buy) — 현재 강점의 완성

#### A1. 폰 추천 마법사 `/finder`
- 5문항 (예산 / 용도: 카메라·게임·라이트 / 크기 / 브랜드 / 신품·중고 허용)
- → 기존 verdict + 지표로 **상위 3종 + 각각의 이유** 출력. 신규 데이터 불필요, 기존 89종 DB 재활용
- SEO: "60만원대 폰 추천", "부모님 폰 추천" 등 결과 프리셋을 정적 페이지화

#### A2. TCO(총소유비용) 계산기 `/tco/[model]`
- **구매가 − N년 후 예상 중고가 + 예상 수리 리스크(보험료)** = 월 실질 비용
- "아이폰 16, 월 3.1만원에 쓰는 셈" — 국내에 이 지표 제공하는 곳 없음. **시그니처 지표 후보**
- 필요 데이터: 이미 보유 (출시가·시세 시계열·수리비). 감가 곡선만 시리즈별 회귀로 모델링

#### A3. 구매 루트 계산기 `/route-calc/[model]`
- 자급제+알뜰폰 vs 통신사 공시지원금 vs 선택약정 25% — 24개월 총액 비교
- 데이터: 스마트초이스(smartchoice.or.kr) 공시지원금 — 준공식, 조회 가능
- 위약금(잔여 할인반환금) 계산기 포함

#### A4. 중고 적정가 판독기 `/price-check`
- 모델 + 상태(S/A/B) + 배터리 효율 입력 → 적정가 범위 + "이 매물 비쌈/적정/급처" 판정
- 번개장터 시세 파이프라인(자동화 2순위)의 첫 소비처

#### A5. 타이밍 인텔리전스 `/when-to-buy/[model]`
- 시리즈별 역사적 패턴: "신형 발표 후 4~8주에 구형 시세 평균 −12%"
- 출시 캘린더(루머 아닌 **공식 발표·역대 패턴 기반 예측**, 신뢰등급 표기)와 결합
- → "지금 사지 말고 9월 이후" 같은 **행동 지시형** 콘텐츠. 재방문·공유 유발

### 🅑 쓰는 중 (Own) — 최대 미개척지, P2 공략

#### B1. 내 폰 대시보드 `/my` (개인화의 핵심)
- 기기 등록(모델+구매일, 로컬스토리지로 시작 → 계정은 후순위)
- 한 화면에: 보안지원 D-day / 현 시세 & 이번 달 변동 / 미해결 이슈 / 잔존가치 곡선 위 내 위치 / "지금 팔면 vs 1년 더 쓰면" 손익
- **브랜드 락인 장치 1호.** 모든 알림·뉴스레터의 근간

#### B2. 케어 손익 계산기 `/care-calc/[model]`
- 삼성케어플러스 vs 애플케어+ vs 무보험: 보험료 총액 vs (파손확률 × 수리비) 기댓값
- 이미 `withCareKRW` 필드 보유 — 데이터 90% 준비됨
- "이 기종은 화면 수리비가 시세의 70%라 케어 가입이 유리" 식 기종별 판정

#### B3. 배터리 교체 vs 기기 교체 판정기
- 배터리 교체비(보유) + 잔여 지원기간(보유) + 현 시세(보유) → "교체하고 2년 더" vs "지금 파는 게 이득"
- 검색량 큰 질문("아이폰 배터리 교체 가격 할까 말까")을 정확히 타격

#### B4. 업데이트 트래커 `/updates`
- 기종별 최신 OS/보안패치 버전·배포일 (endoflife.date + 삼성 보안공지 자동 수집)
- One UI/iOS 메이저 업데이트 지원 기종 리스트 — 발표 시즌마다 트래픽 스파이크 자산

#### B5. 이슈 센터 고도화 `/issues`
- 전 기종 이슈 통합 피드 + 심각도/상태 필터 + **무상수리·리콜 전용 페이지** (검색 수요 큼)
- 사용자 제보 폼 → 편집 검수 큐 → 게재 (UGC는 받되 게시는 사람이 — 기존 원칙 유지)
- "이 기종 이슈 알림 받기" 구독 (B1과 연동)

### 🅒 팔 때 (Sell) — 시세 데이터 재활용, 저비용 고효율

#### C1. 판매 타이밍 알림
- "내 기기가 ○○원 아래로 떨어지면 / 신형 발표 6주 전이 되면 알림"
- 이메일부터 시작 (Resend 등), 웹푸시는 후순위

#### C2. 처분 루트 비교 `/sell/[model]`
- 중고 직거래(번개 시세) vs 트레이드인(삼성/애플 보상판매 고시가) vs 중고매입업체(민팃 등) — 3루트 실수령액 비교
- 트레이드인 고시가는 공식 페이지에서 조회 가능 → 반자동화

#### C3. 판매 준비 체크리스트 (기존 used-check의 판매자 버전)
- 초기화·계정 해제(FMI/FRP)·유심·부속 — 기종별. 인쇄/공유 가능한 포맷

### 🅓 고르기 (Compare/Explore) — 기존 기능 고도화

#### D1. 자유 비교 빌더 `/compare?a=x&b=y(&c=z)`
- 큐레이션 35쌍 → 임의 2~3종 비교. 정적 페이지는 유지(SEO), 빌더는 클라이언트 조합
#### D2. 랭킹 허브 `/best`
- "잔존가치 방어 TOP10", "지원기간 긴 순", "수리비 부담 낮은 순", "예산 30/50/80/100만 최강" — 전부 기존 지표의 정렬 뷰 = **제작비 0에 가까운 SEO 페이지 수십 장**
#### D3. 시리즈 허브 `/series/[갤럭시S|아이폰|Z플립…]`
- 세대별 타임라인 + 세대간 "건너뛰어도 되나" 판정

## 4. 데이터 & 자동화 로드맵 (automation-plan.md 승계·확장)

```
Phase A (즉시, 2주) — 신선도 확보
  ① endoflife.date 주1회 Cron → eol 필드 자동 갱신     [완전 자동, 확인됨]
  ② 번개장터 find_v2 일1회 → resale 시계열 적재          [완전 자동, 확인됨]
  ③ 모든 페이지에 "데이터 기준일" 뱃지 노출 강화

Phase B (1~2개월) — 데이터 레이어 승격
  ④ JSON → SQLite(Turso)/Postgres. 시세는 시계열 테이블 분리
  ⑤ 스마트초이스 공시지원금 수집 → 구매 루트 계산기
  ⑥ 트레이드인 고시가 반자동 수집 → 처분 루트 비교

Phase C (분기) — 인텔리전스
  ⑦ 시리즈별 감가 곡선 모델 → TCO·타이밍 예측
  ⑧ 이슈 키워드 모니터링 → 편집 큐 (게시는 사람)
  ⑨ 데이터 이상 감지 (시세 급변·엔드포인트 변경 알림)
```

**신뢰 인프라 (브랜드 자산):** 모든 수치에 `source·asOf·confidence(공식/준공식/추정)` 3종 표기(현 스키마 유지) + `/about/data` 데이터 방법론 공개 페이지 + 정정 로그. → "폰덱스 수치는 출처가 있다"를 브랜드 인식으로.

## 5. 성장 & SEO 전략

### 프로그래매틱 SEO (기존 구조의 곱셈)
- 현재: 89종 × 7페이지 ≈ 620장 → 확장 후: **(기종 × 문서) + (기종 × 질문형) + 랭킹 + 시리즈 ≈ 2,500장+**
- 질문형 랜딩: `"{기종} 지금 사도 될까"`, `"{기종} 언제까지 지원"`, `"{기종} 배터리 교체 가격"` — 각각 이미 있는 데이터의 재편집. 한국 검색 실수요 그대로 타격
- 네이버 대응: 서치어드바이저 등록, 네이버 인플루언서/블로그 요약판 배포 (국내 트래픽의 절반은 네이버)

### 재방문 엔진
- **주간 뉴스레터 "이번 주 폰 시장"**: 시세 등락 TOP5 · 새 이슈 · EOL 임박 기종 — 전부 자동화 데이터에서 생성
- 알림 3종: 시세 도달 / 이슈 발생 / 지원종료 D-90
- 월간 리포트 `/report/2026-07`: "7월 중고폰 시세 리포트" — 언론 인용 유도용 (백링크 자산)

### 배포 채널
- 유튜브 쇼츠/릴스: "이 폰 지금 사도 될까 30초" — verdict 화면 그대로 영상화 (템플릿 자동 생성 가능)
- 커뮤니티 시딩: 뽐뿌·클리앙에 리포트 공유 (광고 아닌 데이터 제공 포지션)

## 7. 기술 로드맵

- **현행 유지**: Next.js 정적 생성 + ISR. 자동화는 Vercel Cron + 데이터 레이어만 교체 (프레젠테이션 무수정 — 기존 설계의 강점)
- **추가 필요**:
  - DB(Turso/Postgres) + Drizzle — Phase B
  - 검색: 89종은 클라이언트 필터로 충분 → 300종+ 시 Pagefind/Algolia
  - 알림: Resend(이메일) → 웹푸시(후순위)
  - 계정: 로컬스토리지 우선 → 이메일 매직링크 (알림 구독 시점에만 요구)
  - 분석: Plausible/GA4 + 판정 노출→클릭 퍼널 측정
- **품질 게이트**: 데이터 스키마 CI 검증(Zod, 이미 존재) + 링크 데드체크 주1회 + Lighthouse 예산
